Branching and Merging

Branching is the act of creating multiple copies of similar code in the same repository. When code is fine-tuned and tested in the branch, and we know it works, we can add it into master. This process is called merging. I have learned how to:

  • Create a branch
  • Push newly created feature branch to my repository
  • Switch between master branch and feature branches
  • Merge feature branch to master branch
  • Deleting feature branches
